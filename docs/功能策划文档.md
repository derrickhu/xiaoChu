## 二、核心战斗系统（转珠规则）

### 2.1 棋盘设定

| 项目 | 设定值 |
|------|--------|
| 棋盘尺寸 | 5行 × 6列 = 30 颗珠子 |
| 珠子类型 | 金、木、水、火、土、心（回血） |
| 操作方式 | 手指按住珠子拖动，沿途珠子八方向即时交换 |
| 操作限时 | 8 秒（可被 buff/法宝延长） |
| 回合结束 | 松手 = 回合结束 → 消除结算 → 怪物攻击 1 次 |

### 2.2 消除规则

| 消除数 | 基础倍率 | 特殊效果 |
|--------|----------|----------|
| 3 消 | ×1.0 | 对应属性宠物单体攻击 |
| 4 消 | ×1.5 | 对应属性宠物单体攻击 |
| 5 消及以上 | ×2.0 | **全体攻击 + 眩晕怪物 1 回合**（受 `stunDurBonus` 加成） |

**关键规则**：
- 无对应属性宠物的珠子消除**不计入 Combo**，消除无伤害效果
- 封印的珠子无法被拖动，但可以被消除
- 消除动画逐组播放（每组 16 帧），下落填充后自动检测连消
- 每次消除（每组）Combo+1，多组依次递增

### 2.3 Combo 加成系统

Combo 采用**递减阶梯**倍率，避免后期数值爆炸：

| Combo 段数 | 每段加成 | 累计倍率公式 |
|-----------|----------|--------------|
| 1~8 段 | +35%/段 | `1 + (combo-1) × 0.35` |
| 9~12 段 | +20%/段 | 在 8 段基础上 `+ (combo-8) × 0.20` |
| 13+ 段 | +10%/段 | 在 12 段基础上 `+ (combo-12) × 0.10` |

**Combo 倍率示例**：

| Combo | 倍率 |
|-------|------|
| 1 | 1.00× |
| 3 | 1.70× |
| 5 | 2.40× |
| 8 | 3.45× |
| 12 | 4.25× |
| 16 | 4.65× |

**Combo 里程碑特效**（5/8/12/16 连击时触发视觉特效和音效，不影响数值）

### 2.4 心珠回血

- 心珠消除与属性无关，直接回血
- 基础回复量：`(12 + floor × 0.8) × 消除倍率`
- 受 `heartBoostPct` 加成、法宝 `heartBoost` 加成
- 受 Combo 倍率加成（在 `petAtkShow` 阶段统一乘以 Combo 倍率）
- 可被怪物技能 `healBlock`（噬灵术）减半

### 2.5 转珠操作细节

- **八方向移动**：上、下、左、右、左上、右上、左下、右下
- **即时交换**：手指移动到相邻格时立即交换，无路径限制
- **可连续换位**：按住不松手可连续移动，沿途珠子依次交换
- **封印珠不可拖动**：被封印的珠子不能作为起始拖动对象，但可以被交换到
- **松手即结算**：松手后立即进入消除检测流程

## 三、五行克制系统

### 3.1 克制关系

```
金 → 木 → 土 → 水 → 火 → 金（循环相克）
```

| 攻击方 | 被克方 | 倍率 |
|--------|--------|------|
| 金 | 木 | ×2.5 |
| 木 | 土 | ×2.5 |
| 土 | 水 | ×2.5 |
| 水 | 火 | ×2.5 |
| 火 | 金 | ×2.5 |
| 被克方向 | - | ×0.5 |

克制伤害受 `counterDmgPct` 加成；触发克制时有**闪光特效**和**克制！文字提示**。

### 3.2 灵珠权重机制

棋盘初始化和下落填充时，各属性珠子按权重概率出现：

| 属性 | 基础权重 | 条件加成 |
|------|----------|----------|
| 五行属性 | 各 1.0 | 敌人属性 → 该属性 ×1.4 |
| 心珠 | 0.8 | - |
| 法宝 beadRateUp | - | 指定属性 ×1.5 |
| goodBeadsNextTurn | - | 上场宠物对应属性 ×1.5 |

## 四、Roguelike 局内流程

### 4.1 流程总览

```
标题画面
  ├── 开始挑战 → startRun → nextFloor → 事件循环
  └── 继续挑战 → resumeRun → 事件循环

事件循环（每层）：
  ┌─ 事件页面（展示本层事件）
  │   ├── 战斗类（普通/精英/BOSS）→ 战前准备 → 转珠战斗 → 胜利奖励 → nextFloor
  │   ├── 奇遇 → 应用效果 → nextFloor
  │   ├── 商店 → 选择购买 → nextFloor
  │   └── 休息 → 回血/强化 → nextFloor
  │
  ├── floor > 30 → 通关结算
  └── heroHp <= 0 → 失败/复活
```

### 4.2 塔层总览

| 项目 | 设定 |
|------|------|
| 总层数 | 30 层 |
| 初始血量 | 100 HP |
| 上场宠物 | 固定 5 只（金木水火土各 1 只） |
| 装备法宝 | 1 件（全队被动效果） |
| 通关条件 | 通过第 30 层 BOSS |
| 失败条件 | 英雄血量归零且无复活手段 |

### 4.3 事件生成规则

**固定事件**：
- 每 10 层（10/20/30）：强制 **BOSS** 战
- 第 5 层：强制 **精英** 战（保证前期获得法宝机会）

**随机事件权重**（基础值）：

| 事件类型 | 基础权重 | 说明 |
|----------|----------|------|
| 普通战斗 | 70 | 15 层后 -10，22 层后再 -10 |
| 精英战斗 | 8 | 随层数递增，5 的倍数层大幅增加 |
| 奇遇 | 5 | 随层数缓慢递增 |
| 商店 | 4 | 随层数缓慢递增 |
| 休息 | 3 | 随层数缓慢递增 |

**特殊规则**：
- 前 2 层：只出普通战斗
- 3~4 层：开放奇遇和休息，不出精英和商店
- 5 层起：全面开放

### 4.4 修仙境界系统

每层通关后自动提升修仙境界，增加血量上限（不回血）：

| 层数范围 | 境界 | 每层血量上限增加 |
|----------|------|-----------------|
| 1 | 凡人 | +0 |
| 2~4 | 感气期→凝气初成 | +8 |
| 5 | 炼气一层 | +11 |
| 6~9 | 炼气二~五层 | +10 |
| 10 | 筑基初期 | +15 |
| 11~14 | 筑基中期→开光初期 | +13 |
| 15 | 开光圆满 | +18 |
| 16~19 | 融合初期→心动初期 | +15 |
| 20 | 心动圆满 | +20 |
| 21~24 | 金丹初期→金丹圆满 | +18 |
| 25 | 元婴初期 | +23 |
| 26~29 | 元婴中期→化神初期 | +20 |
| 30 | 化神圆满 | +25 |

**隐性攻击加成**：每过 5 层（第 6/11/16/21/26 层）自动获得全队攻击加成（12%→14%→16%→18%→20%）。

### 4.5 战后奖励规则

胜利后进入**三选一**（速通为四选一）奖励界面：

| 敌人类型 | 奖励组合 | 速通额外奖励 |
|----------|----------|-------------|
| 普通战斗 | 2只灵兽 + 1件法宝 | +1只灵兽 |
| 精英战斗 | 1只灵兽 + 1件法宝 + 1个中档buff | +1件法宝 |
| BOSS战 | 1只灵兽 + 1件法宝 + 1个大档buff | +1件法宝 |

**速通判定**：5 回合内击败敌人。

**宠物获取机制**：
- 所有宠物从本局宠物池（`sessionPetPool`，每属性 5 只共 25 只）中抽取
- 偏向抽取已拥有的宠物以促进升星
- 已满星（3星）的宠物从候选池中排除

## 五、宠物系统框架

> 详细的 100 只宠物配置（技能、攻击、CD等）见单独的宠物梳理文档

### 5.1 总览

| 项目 | 设定 |
|------|------|
| 总数 | 5 属性 × 20 只 = 100 只 |
| 上场数 | 固定 5 只（每属性至少 1 只） |
| 星级 | 1~3 星（★1 无技能，★2 有技能，★3 强化技能） |
| 档位 | T1（基础）、T2（中等）、T3（稀有） |
| 攻击来源 | 仅消除对应属性珠时攻击 |

### 5.2 星级与升星

| 星级 | 获取方式 | 技能状态 | 攻击加成 |
|------|----------|----------|----------|
| ★1 | 奖励/奇遇/商店获得 | 无技能 | 基础攻击 |
| ★2 | 再次获得同一只（重复获取自动升星） | 解锁主动技能 | 攻击 ×1.3 |
| ★3 | 第三次获得同一只 | 技能强化（部分有特殊强化效果） | 攻击 ×1.7 |

★3 触发**满星庆祝特效**，并记录到图鉴。

### 5.3 技能 CD 机制

- 每完成 1 次转珠回合 → 所有宠物技能 CD-1
- CD = 0 → 头像亮起闪光提示，**转珠前**上滑/点击释放
- 释放后 CD 重置为初始值，重新冷却
- `skillCdReducePct` 加成可减少实际 CD

### 5.4 技能类型分类

| 大类 | 子类型举例 | 说明 |
|------|-----------|------|
| 增伤 | dmgBoost, allDmgUp, comboDmgUp | 提升自身/全队伤害 |
| 变珠 | convertBead, convertRow, convertCol, convertCross, replaceBeads | 改变棋盘珠子属性 |
| 防御 | shield, shieldPlus, dmgImmune | 获得护盾或减伤 |
| 控制 | stun, stunDot | 眩晕敌人 |
| 回复 | healPct, healFlat, fullHeal | 回复血量 |
| 直伤 | instantDmg, multiHit, teamAttack | 跳过消除直接造成伤害 |
| 辅助 | comboPlus, comboNeverBreak, critBoost, goodBeads | Combo/暴击/珠子辅助 |

## 十一、复活机制

玩家血量归零时，按以下优先级检测复活条件：

| 优先级 | 来源 | 效果 | 次数 |
|--------|------|------|------|
| 1 | 临时复活（奇遇/buff `tempRevive`） | 恢复 30% 血量 | 1 次 |
| 2 | 额外复活（runBuff `extraRevive`） | 恢复 25% 血量 | 可叠加 |
| 3 | 法宝不灭金身（weapon.type='revive'） | 恢复 20% 血量 | 每局 1 次 |
| 4 | 分享复活（`adReviveOffer`） | **满血恢复** + 清除负面 buff | 每局 1 次 |
| - | 以上均不可用 | 进入失败结算 | - |

**分享复活**：调用 `wx.shareAppMessage` 后直接给予复活（微信不保证回调可靠性，为保证体验直接放行）。

## 十二、排行榜系统

### 12.1 三个榜单

| 榜单 | 排序依据 | 展示内容 |
|------|----------|----------|
| 速通榜 | 层数 DESC → 回合数 ASC | 头像、昵称、层数、回合数、阵容 |
| 图鉴榜 | 图鉴收集数 DESC | 头像、昵称、收集数量 |
| 连击榜 | 最高连击数 DESC | 头像、昵称、最高 Combo |

### 12.2 数据提交

- **提交时机**：每局结束时自动提交
- **前提条件**：云环境就绪 + 用户已授权（头像昵称）
- **预热机制**：启动时后台静默拉取，保持云函数实例热状态

### 12.3 用户授权

- 首次点击排行按钮时弹出微信授权请求（获取头像和昵称）
- 授权后缓存到本地，后续自动使用
- 未授权可以游客身份进入排行榜（仅查看，不上榜）

## 十三、图鉴系统

### 13.1 收集规则

- 宠物升至 **★3**（满星）时自动记录到图鉴
- 图鉴按 5 属性分类，每属性 20 只
- 未收集的宠物显示为锁定状态
- 已收集但未查看详情的宠物显示**红点**标记

### 13.2 "带它出战"功能

- 在图鉴中选择已收集的宠物，点击"带它出战"
- 下一局开始时，该宠物以 **★1 形态**替换初始队伍中同属性的宠物
- 若队伍中无同属性宠物，替换最后一只

## 十四、新手引导

### 14.1 触发条件

首次游戏时自动触发（检查本地 `tutorialDone` 标记）。

### 14.2 四步试炼

| 步骤 | 主题 | 回合数 | 教学内容 |
|------|------|--------|----------|
| 步骤 0 | 转珠基础 | 1 回合 | 按住珠子沿路径拖动，沿途珠子依次交换，松手后三连消除 |
| 步骤 1 | Combo 与心珠 | 2 回合 | 回合1: 一次消除多组触发 Combo；回合2: 被打伤后消心珠回血 |
| 步骤 2 | 五行克制 | 2 回合 | 回合1: 用克制属性造成高伤害；回合2: 用被克属性感受伤害减半 |
| 步骤 3 | 宠物技能 | 1 回合 | 上滑宠物头像释放技能 |

**教学设计特点**：
- 每步使用**固定棋盘布局**，保证引导确定性
- 配置弱敌（极低HP/无攻击），避免教学中失败
- 引导箭头标示拖拽起点和路径
- 教学中锁定非引导区域操作
- 教学完成后正式进入第 1 层并赠送初始法宝

## 十六、数值框架概要

### 16.1 怪物数值分段

怪物数值按 6 个层段定义，层段内线性插值并加 ±15% 随机波动：

| 层段 | HP 范围 | ATK 范围 |
|------|---------|----------|
| 1~5 | 200~360 | 8~14 |
| 6~10 | 380~650 | 14~22 |
| 11~15 | 680~1100 | 20~32 |
| 16~20 | 1100~1700 | 30~46 |
| 21~25 | 1600~2400 | 40~58 |
| 26~30 | 2200~3000 | 50~68 |

**防御**：`def = round(atk × 0.35)`

### 16.2 精英/BOSS 倍率

| 类型 | HP 倍率 | ATK 倍率 | DEF 倍率 | 技能数 |
|------|---------|----------|----------|--------|
| 精英 | ×2.2~2.8 | ×1.6~2.0 | ×1.5 | 2 个 |
| BOSS(10层) | ×2.5 | ×1.5 | ×1.2 | 2 个 |
| BOSS(20层) | ×3.3 | ×1.8 | ×1.4 | 2 个 |
| BOSS(30层) | ×4.1 | ×2.1 | ×1.6 | 3 个 |

### 16.3 伤害公式概要

```
最终伤害 = 基础攻击 × 消除倍率 × Combo倍率 × 全局加成 × 属性加成
           × 五行克制 × 暴击 × 残血爆发 × 法宝加成 - 敌人防御
```

### 16.4 暴击系统

| 项目 | 默认值 | 加成来源 |
|------|--------|----------|
| 暴击率 | 0% | 宠物技能 critBoost、法宝 critAll/comboToCrit/guaranteeCrit |
| 暴击伤害 | +50% | 宠物技能 critDmgUp、法宝 critAll |
| 暴击率上限 | 100% | - |

### 16.5 残血爆发

| 血量比例 | 伤害倍率 |
|----------|----------|
| > 30% | ×1.0 |
| ≤ 30% | ×1.5 |
| ≤ 15% | ×2.0 |

### 16.6 怪物技能触发

- 普通怪：前 5 层 1 个轻量技能，6~8 层 1 个基础技能，9 层起逐步增加
- 18 层起可能有 2 个技能
- 25 层以上 30% 概率带 3 个技能
- 精英：必带 2 个技能（从 10 种中随机不重复抽取）
- BOSS：有专属技能组（2~3 个）

### 16.7 怪物技能类型

| 类型 | 说明 |
|------|------|
| 增强类 | atkBuff（攻击提升） |
| 持续伤害 | poison（瘴毒）、bossDot（业火） |
| 控制类 | seal/sealRow/sealAttr/sealAll（封珠）、stun（眩晕） |
| 干扰类 | convert（灵脉紊乱）、breakBead（碎珠术） |
| 直伤类 | aoe（横扫）、bossBlitz（连击）、bossAnnihil（灭世） |
| 削弱类 | defDown（碎甲）、bossWeaken（双降）、healBlock（噬灵） |
| 回复类 | selfHeal（自愈）、bossDrain（吸血）、bossDevour（噬魂+窃取治疗） |
| 反伤类 | bossMirror（反弹伤害） |
| 终极类 | bossUltimate（超越·终焉：大伤害+全封珠+眩晕） |
