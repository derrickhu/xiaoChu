# 《龙珠战纪》微信小游戏开发策划案（程序落地版·玩法重设计）

# 一、核心玩法总纲（全新设计，脱离原有框架）

核心原则：单主角设定，取消原有英雄系统，替换为“装备技能体系”，以“三消打怪”为核心玩法，结合“装备佩戴-属性球消除-技能触发”的联动逻辑，搭配局内随机掉落装备/技能的即时使用机制，兼顾操作趣味性与轻量化体验；全程采用微信小游戏云开发模式（云数据库+云函数），无需自行搭建服务器，降低开发成本，保障数据安全与跨设备同步。

核心闭环：主角进入关卡→佩戴6类装备（对应不同属性技能）→三消对应属性宝珠→触发装备关联技能→击败敌方通关→局内随机掉落装备/技能（即时使用）→提升战力挑战更高关卡，形成“闯关-获取装备-强化战力-进阶挑战”的核心循环。

# 二、核心系统设计（全新搭建，适配单主角+装备技能逻辑）

## 2.1 单主角设定（唯一可控角色，简化操作）

主角为原创固定角色，不贴合任何IP，立绘风格参考《杖剑传说》，采用简洁线条、鲜活色彩设计，造型简约大气，适配微信小游戏轻量化视觉需求；主角无角色解锁/养成，战力完全依赖装备佩戴与技能搭配，简化数值体系，聚焦装备与技能策略组合，降低开发复杂度，同时让玩家专注于三消操作与技能触发节奏。

- 主角基础属性：固定初始ATK（攻击）、HP（血量），无等级提升，属性提升仅通过装备的被动技能实现，无需复杂数值计算，所有基础数据存储在云数据库“playerData”集合，关联玩家微信openid。

- 主角核心机制：全程唯一可控，战斗中仅通过三消宝珠触发装备技能，自身无主动攻击能力，所有输出与生存依赖装备技能，简化操作逻辑，适配微信小游戏轻量化需求。

- 数据存储：主角基础属性、当前佩戴装备、已拥有装备/技能列表，统一存储在云数据库“playerData”集合，实时同步，支持跨设备登录查看与操作。

## 2.2 装备系统（核心替代英雄系统，关联技能与属性）

核心设定：共6类装备，分别为武器、铠甲、战靴、披风、头盔、饰品，每类装备仅可佩戴1件；每类装备可扩展多种具体装备，每种具体装备对应一种宝珠属性（火、水、木、光、暗、心），佩戴后解锁对应属性的技能（普通技能+绝技+被动技能）。为避免同名称装备差异化过大，每个装备可随机到的技能类型、数值均设定明确范围限制（同名称、同品质装备的技能数值处于同一区间），技能触发与三消对应属性宝珠绑定，形成“装备-属性-技能”的联动，所有装备数据存储在云数据库“equipmentData”集合。

### 2.2.2 技能规则（与三消操作、装备绑定）

- 普通技能：佩戴对应类别任意具体装备后解锁，触发条件为“消除3个及以上该装备对应属性宝珠”，触发后自动生效，无冷却时间，每次消除满足条件即可触发。若佩戴多个同属性装备（不同类别），则所有同属性装备的普通技能会在同一回合同步生效。技能名称、伤害/效果数值支持随机组合，但均有明确范围限制（同名称、同品质装备的数值区间一致，如普通品质火焰长剑普通技能伤害固定在100-150，稀有品质固定在200-280），所有随机组合规则与数值范围存储在云数据库“equipmentSkillRule”集合，确保同属性、同名称装备差异可控，同时保持适度多样性。

- 绝技：佩戴对应类别任意具体装备后解锁，激活条件为“该装备累计释放普通技能达到指定次数”（次数范围为3-5次，同品质装备次数一致，可随机浮动，如普通品质固定3次，传说品质固定5次）；激活后，玩家可选择在任意一回合主动触发，触发后该装备的普通技能释放次数统计清零，需重新累计次数才能再次激活绝技。绝技效果为同装备普通技能的3-5倍（倍数随品质递增，数值在固定区间内随机浮动，同品质区间一致），技能名称、具体效果随机组合但有范围限制，兼顾属性一致性与差异可控性。

- 被动技能：佩戴对应类别任意装备后自动激活，无需触发，持续生效，且不产生任何伤害效果。效果类型固定为“提升主角HP/ATK、增加主角防御、降低敌方ATK/防御、缩短对应属性技能累计次数”4类，每类装备随机抽取2类效果组合，数值随装备品质递增且有明确范围限制（同名称、同品质装备的数值区间一致，如普通品质HP提升固定100-200，传说品质提升固定500-800），确保同类别、同名称装备被动效果差异合理，提升策略搭配空间的同时避免失衡。

- 技能属性：技能属性与装备类别对应属性一致，存在属性克制（火克木、木克水、水克火、光克暗、暗克光、心属性无克制，仅用于回血/辅助），克制时伤害提升50%（数值可微调，随机浮动±5%），克制关系内置在战斗逻辑中，技能数值随机组合后自动适配克制加成规则。

### 2.2.3 装备品质与获取（适配局内掉落+长期收集）

- 装备品质：分为普通（白色）、稀有（蓝色）、史诗（紫色）、传说（橙色）4个等级，品质越高，技能效果越强。普通技能伤害、绝技倍数、被动技能加成均随品质递增，且每个品质对应固定的数值范围，同名称、同品质装备的技能数值均处于对应区间内，避免同名称装备差异化过大。支持“同名称不同品质”区分（如“火焰长剑”可分为普通、稀有、史诗、传说4种品质），同名称不同品质装备的核心技能类型一致，数值按品质区间递增，立绘细节也有区分（如传说品质装备立绘带光泽特效）；同一类别、不同属性的装备，品质判定标准与数值范围规则一致。

- 获取途径：1. 局内随机掉落（核心途径）：每局关卡战斗中，击败敌方后有概率掉落任意类别、任意属性、任意品质的装备，掉落装备的名称、技能组合、数值均随机生成（遵循对应类别、属性规则），可即时佩戴使用（仅当前局有效）；2. 关卡通关奖励：通关指定关卡可获得固定类别、随机属性、随机名称与技能组合的装备（永久拥有），品质随关卡难度提升，奖励数据存储在云数据库“levelReward”集合；3. 每日任务/成就奖励：完成每日任务、达成成就可获得稀有/史诗品质装备（永久拥有），类别、属性、名称、技能组合随机，奖励领取状态同步至云端。

- 装备管理：玩家拥有的永久装备存储在云数据库“playerData”集合的“ownEquipment”字段中，每个装备新增“drawId”（立绘唯一标识），支持佩戴/卸下操作；佩戴后，角色立绘将同步显示该装备的穿戴效果，多件装备佩戴时，立绘自动叠加显示（如同时佩戴火焰长剑、流水铠甲，角色立绘将同时呈现武器与铠甲效果）。装备卸下后，技能、被动效果及立绘穿戴效果同步消失，操作记录实时同步至云端，避免数据丢失。

- 立绘相关：每个装备对应独立立绘（drawId唯一），立绘风格参考《杖剑传说》，采用线条简洁、颜色鲜活的设计，与装备类别、品质匹配（如火焰类装备以红橙为主色调，光辉类以亮黄、浅蓝为主色调）；立绘支持在角色立绘上真实穿戴，穿戴逻辑内置在角色渲染模块，根据玩家当前佩戴的6类装备（每类1件），自动拼接装备立绘与角色基础立绘，形成完整的角色穿戴效果。角色立绘为原创设计，不贴合任何IP，线条简洁流畅、色彩明亮鲜活，适配整体视觉风格，提升玩家视觉体验。

## 2.3 三消核心玩法（战斗核心，联动装备技能）

核心逻辑：战斗界面为5×6宝珠矩阵，宝珠分为6种（对应6类装备/属性），玩家通过滑动屏幕交换相邻宝珠，形成3个及以上同色宝珠即可消除，消除对应属性宝珠触发已佩戴的对应装备技能，击败敌方即可通关，战斗数据实时同步至云数据库。

- 战斗流程：1. 进入关卡后，玩家选择已拥有的6类装备（每类1件）进行佩戴，确认后进入战斗；2. 敌方出现（单波/多波刷新，根据关卡难度设定），玩家进行三消操作，消除宝珠触发对应技能，消耗敌方血量；3. 每回合玩家可进行1次三消操作，敌方回合自动攻击主角，主角血量为0则战斗失败；4. 击败所有敌方后通关，通关后结算奖励（装备、道具），同时记录关卡进度；5. 战斗过程中，击败敌方有概率掉落装备/技能，掉落物品即时弹出，玩家可选择“立即使用”（装备即时佩戴、技能即时触发）或“暂存”（仅当前局有效，战斗结束后失效）。

- 三消规则：支持横向、纵向消除，3个同色宝珠消除触发普通技能，4个同色宝珠消除触发普通技能+额外伤害，5个同色宝珠（或特殊形状，如T型、L型）消除触发装备掉落；Combo连续消除（多组宝珠连续消除）可叠加技能伤害（每增加1个Combo，伤害提升10%），Combo数据实时计算并渲染。

- 局内掉落即时使用规则：掉落装备可即时替换当前佩戴的同类别装备，替换后技能与被动效果立即更新；掉落技能为“临时技能”（仅当前局有效），可直接点击使用，触发对应属性的额外效果（如全屏清场、瞬间回血），使用后消失，掉落记录与使用状态实时同步至本地，无需存入云端（仅当前局有效）。

## 2.4 关卡系统（适配单主角+装备技能玩法，轻量化设计）

核心原则：关卡按“难度递进+主题区分”设计，无需复用原有地图资源，全新设计贴合龙珠IP的关卡场景，聚焦“三消操作+装备技能搭配”，每个关卡设定不同的敌方配置与通关条件，所有关卡进度、通关记录存储在云数据库“levelProgress”集合。

- 关卡分层：分为普通、困难、极难3个难度，普通难度解锁所有基础关卡，通关普通难度解锁困难难度，通关困难难度解锁极难难度，难度提升主要体现为“敌方HP/ATK提升、敌方技能增强、宝珠刷新概率调整”，无需新增关卡地图，仅修改敌方与宝珠配置。

- 关卡主题：按属性分为6个主题关卡（对应6类装备属性），每个主题关卡包含10个小关卡，主题关卡内的宝珠以对应属性为主，敌方也以对应属性为主，引导玩家搭配对应装备挑战，提升玩法针对性；此外新增“混合属性关卡”（所有属性宝珠随机刷新、敌方为混合属性），作为高阶挑战。

- 通关条件与奖励：单关卡通关条件为“击败所有敌方”，部分关卡新增特殊条件（如“在10回合内通关”“触发5次Combo”），达成特殊条件可获得额外奖励；通关奖励包括永久装备、技能道具（用于强化装备技能），普通难度掉落普通/稀有装备，困难难度掉落稀有/史诗装备，极难难度掉落史诗/传说装备，奖励发放后实时同步至云数据库玩家账户。

## 2.5 辅助玩法（提升长期留存，适配云开发）

### 2.5.1 每日任务（提升每日活跃度）

轻量化设计，通过云函数每日0点定时重置，任务目标围绕核心玩法设计，奖励以装备、技能道具为主，引导玩家每日登录，所有任务进度、奖励领取状态存储在云数据库“dailyTask”集合。

- 固定3个每日任务：1. 完成1次任意关卡挑战（奖励：普通装备碎片×3）；2. 触发10次普通技能（奖励：技能道具×1）；3. 局内掉落并使用1件装备（奖励：稀有装备碎片×1）；完成所有任务可额外获得稀有装备抽奖机会×1。

- 重置机制：通过云函数定时触发，无需依赖本地时间，避免设备时间偏差，任务进度跨设备同步，奖励领取后即时更新云端数据，避免重复领取。

### 2.5.2 成就系统（激励长期探索）

覆盖三消操作、装备收集、关卡挑战等维度，完成成就可获得稀有装备、顶级技能道具，所有成就数据（完成状态、进度、奖励领取情况）存储在云数据库“achievementData”集合，关联玩家openid。

- 核心成就分类：1. 操作类（单次Combo≥8、连续触发3次绝技、单局消除100个宝珠）；2. 装备类（收集10件稀有装备、佩戴全套传说装备、解锁所有装备技能）；3. 关卡类（通关所有普通/困难/极难关卡、单局1回合通关、无技能触发通关）；4. 收集类（局内掉落50件装备、使用10次临时技能）。

### 2.5.3 排行榜（低成本社交，提升粘性）

采用“本地缓存+云数据库同步”模式，无需复杂服务器开发，记录玩家核心成绩，支持全服排名对比，满足玩家攀比心理，激励反复挑战，所有排名数据存储在云数据库“rankingData”集合。

- 排行榜分类：1. 关卡排行榜（单关最短通关回合数、单关最高Combo）；2. 技能排行榜（单局触发技能次数、单局绝技触发次数）；3. 装备排行榜（拥有传说装备数量、全套装备品质总和）；仅展示前100名，数据实时同步，每日更新。
