# 《灵宠消消塔》功能版本迭代计划

> 版本日期：2026-03-06

---

## 核心思路

增加长期成长线，从"纯肉鸽"进化为"肉鸽 + 局外养成 + 固定关卡"三支柱结构，提升长期留存。

**三条玩法线**：
- **肉鸽爬塔**（现有）：保持纯粹，不受局外影响，是普通灵宠的来源
- **灵宠池**（新增）：局外持久养成系统，连接肉鸽和固定关卡
- **固定关卡**（新增）：用灵宠池编队挑战，获取高级灵宠和碎片

```
肉鸽模式（保持纯粹）              固定关卡（新增）
  │                                  ▲
  │ 3星图鉴解锁 → ★1入池            │ 灵宠池编队出战
  │ 再次同宠3星 → 碎片              │
  ▼                                  │
          ┌──────────────┐           │
          │   灵宠池     │ ──────────┘
          │ （局外养成） │
          └──────────────┘
              ▲
              │ 高级灵宠掉落 / 碎片
              │
          固定关卡奖励
```

## 灵宠体系

### 两级灵宠

| 类型 | 来源 | 满星 | 星级成长 |
|------|------|------|----------|
| 普通灵宠 | 肉鸽中 3 星图鉴解锁 → ★1 入池 | ★3 | ★1 无技能 → ★2 技能 A → ★3 技能 A 强化 |
| 高级灵宠 | 固定关卡掉落 | ★4 | ★1 无技能 → ★2 技能 A → ★3 技能 A 强化 → ★4 **终极形态**（解锁技能 B 或全新被动） |

### 入池与碎片

| 触发条件 | 结果 |
|----------|------|
| 肉鸽中首次某只宠物达到 ★3 图鉴 | 该宠物 ★1 形态进入灵宠池 |
| 肉鸽中再次同一只宠物达到 ★3 图鉴 | 获得该宠物碎片（用于灵宠池内升星） |
| 固定关卡首通 | 必定掉落一只指定高级灵宠（★1） |
| 固定关卡重复通关 | 碎片 + 概率掉落其他高级灵宠 |

### 碎片与升星

- 灵宠池中的宠物通过碎片升星
- 普通灵宠：升至 ★3 即满星
- 高级灵宠：升至 ★4 才满星，★4 所需碎片量显著高于 ★3（长期目标）
- 碎片按宠物个体区分（A 宠物碎片只能给 A 升星）

### 灵宠池与固定关卡解锁

| 条件 | 效果 |
|------|------|
| 灵宠池中累计 ≥ 5 只宠物 | **解锁固定关卡入口** |
| 池中宠物越多 | 固定关卡编队选择越丰富 |

## Phase 0：前置重构（建议 1~2 周）

> 为后续所有新系统做好架构准备，不改变现有玩法。

### 改动内容

| 改动 | 说明 | 涉及文件 |
|------|------|----------|
| 拆分 GameState | 从 Main 中提取纯状态对象，减轻 Main 职责 | `js/main.js` → 新增 `js/gameState.js` |
| 引入事件总线 | 启用已有的 `tinyemitter.js`，用事件解耦模块通信 | `js/main.js`、各 engine 模块 |
| 存档版本机制 | 为 storage 数据引入 `version` 字段和迁移函数 | `js/data/storage.js` |
| 配置数据外置 | 宠物/法宝/怪物配置从 JS 抽离为 JSON（可选，视工期而定） | `js/data/` 下各文件 |

### 验收标准

- 现有功能完全不变，所有场景正常运行
- Main 类行数降低到 300 行以内
- 新增/修改模块可通过事件订阅而非直接调 `g._xxx()`

## Phase 1：局外经验与永久升级（建议 2~3 周）

### 设计概要

- **保留**现有局内境界系统（每层加血量上限、每 5 层加攻击）不变
- **新增**局外经验系统，经验在局结束后结算，可永久提升基础属性
- 开局属性 = 基础值 + 局外永久加成，局内境界系统在此基础上叠加

### 经验获取

| 来源 | 经验量 | 说明 |
|------|--------|------|
| 消除 | 每组消除 1~3 点（按消除数） | 基础来源 |
| Combo | 每段 Combo 额外 +2 点 | 鼓励高 Combo |
| 击杀敌人 | 5 + floor × 2 | 按层数递增 |
| 通关奖励 | 500 | 通关才有（大头激励） |
| 失败保底 | 上述总和 × 60% | 失败不至于白玩 |

> 具体数值待平衡调优，以上为初始框架。

### 永久升级树（修炼系统）

| 升级项 | 效果 | 每级消耗 | 上限 |
|--------|------|----------|------|
| 体魄 | 开局血量上限 +5 | 递增 | 20 级（+100 HP） |
| 灵力 | 心珠回复基数 +1 | 递增 | 15 级（+15） |
| 悟性 | 初始转珠时间 +0.1s | 递增 | 5 级（+0.5s） |
| 运势 | 奖励出 T2/T3 宠物概率 +1% | 递增 | 10 级（+10%） |
| 亲和 | 初始宠物池 T2 比例 +2% | 递增 | 5 级（+10%） |

> 升级消耗经验递增，保证前期快速感受变化，后期需要大量积累。

### 新增场景

- `cultivation` 场景：修炼升级 UI，从标题页进入
- 展示当前各项等级、经验余额、升级按钮

### 代码改动

| 改动 | 涉及文件 |
|------|----------|
| 战斗中累积经验 | `js/engine/battle.js`（消除/击杀时累加 `g.runExp`） |
| 局结算发放经验 | `js/engine/runManager.js`（endRun 中计算最终经验并存入 storage） |
| 持久化经验和等级 | `js/data/storage.js`（新增 `cultivation` 字段） |
| 开局应用永久加成 | `js/engine/runManager.js`（startRun 中读取 cultivation 修改初始值） |
| 修炼 UI | 新增 `js/views/cultivationView.js` + `screens.js` 中加入入口 |
| 触摸处理 | `js/input/touchHandlers.js`（新增 `tCultivation`） |

## Phase 2：灵宠池（建议 2~3 周）

### 设计概要

- 肉鸽中 ★3 图鉴解锁 → 对应宠物 ★1 入池
- 再次同宠 ★3 → 获得碎片
- 灵宠池内可用碎片升星（普通灵宠满 ★3）
- 灵宠池累计 ≥ 5 只时解锁固定关卡入口

### 灵宠池数据结构

```javascript
// storage.js 中新增持久化字段
{
  petPool: [
    {
      id: 'metal_3',       // 宠物 ID
      attr: 'metal',
      star: 1,             // 当前星级（1~3 普通 / 1~4 高级）
      fragments: 0,        // 当前碎片数
      isAdvanced: false,   // 是否高级灵宠
      source: 'roguelike', // 来源：'roguelike' | 'stage'
    }
  ],
  petPoolUnlockCount: 0,   // 入池总数（用于判断固定关卡解锁）
}
```

### 升星碎片需求

| 目标星级 | 普通灵宠所需碎片 | 高级灵宠所需碎片 |
|----------|-----------------|-----------------|
| ★1 → ★2 | 3 | 5 |
| ★2 → ★3 | 8 | 12 |
| ★3 → ★4 | - | 25 |

> 具体数值待平衡。

### 碎片获取量

| 来源 | 数量 |
|------|------|
| 肉鸽中再次同宠 ★3 图鉴 | 该宠碎片 ×5 |
| 固定关卡通关（普通难度） | 随机碎片 ×2~3 |
| 固定关卡通关（困难难度） | 随机碎片 ×4~6 |
| 固定关卡首通 | 指定高级灵宠 ★1 |

### 新增场景

- `petPool` 场景：灵宠池管理 UI，从标题页进入
- 展示所有已入池灵宠、星级、碎片进度、升星按钮
- 按属性分类筛选
- 点击灵宠查看详情（复用现有宠物详情弹窗）

### 代码改动

| 改动 | 涉及文件 |
|------|----------|
| 3 星图鉴解锁时入池判断 | `js/engine/skills.js`（applyReward 中宠物升 ★3 时检查） |
| 再次 3 星给碎片 | `js/data/storage.js`（addPetDex 扩展逻辑） |
| 灵宠池持久化 | `js/data/storage.js`（新增 petPool 字段 + 云同步） |
| 灵宠池 UI | 新增 `js/views/petPoolView.js` |
| 标题页入口 | `js/views/screens.js`（rTitle 加灵宠池按钮） |
| 固定关卡解锁判定 | `js/data/storage.js`（petPoolUnlockCount >= 5） |

## Phase 3：固定关卡基础版（建议 3~4 周）

### 设计概要

- 灵宠池 ≥ 5 只后解锁
- 包含常驻关卡和周期活动关卡
- 从灵宠池编队 5 只出战（必须凑齐 5 只）
- 复用现有转珠战斗引擎
- 基础版为单波次战斗，通关掉落高级灵宠/碎片

### 关卡配置结构

```javascript
// 新增 js/data/stages.js
{
  id: 'stage_1_1',
  name: '灵山试炼·第一关',
  chapter: 1,               // 章节
  difficulty: 'normal',     // normal | hard | extreme
  type: 'permanent',        // permanent（常驻）| weekly | monthly
  enemies: [                // 敌人配置（单波次，后续扩展多波次）
    { name:'...', attr:'fire', hp:800, atk:25, def:10, skills:['poison','seal'], avatar:'...' }
  ],
  requirements: {           // 编队限制（可选）
    minPets: 5,
    attrLimit: null,        // 后续可加属性限制
  },
  rewards: {
    firstClear: { type:'advancedPet', petId:'adv_fire_1' },  // 首通奖励
    repeatClear: { fragments: { min:2, max:3 }, advancedPetChance: 0.1 },  // 重复奖励
  },
  unlockCondition: { prevStage: null },  // 前置关卡
}
```

### 关卡选择场景

- `stageSelect` 场景：关卡选择界面
- 按章节展示，标注难度、首通状态、推荐战力
- 显示"已解锁"/"未解锁"（前置关卡要求）

### 编队场景

- `stageTeam` 场景：从灵宠池选择 5 只出战
- 复用 `prepareView.js` 的 UI 框架，数据源从 `g.pets/g.petBag` 改为 `storage.petPool`
- 显示每只灵宠的星级、属性、攻击力
- 编队完成后进入战斗

### 战斗流程

- 复用现有 `battle` 场景和战斗引擎
- 差异点：
  - 宠物来自灵宠池编队（非随机）
  - 敌人来自关卡配置（非随机生成）
  - 无法宝系统（基础版暂不支持，后续可扩展）
  - 无奇遇/商店/休息/奖励三选一
  - 胜利后直接进入关卡结算（展示掉落物）
  - 失败后回到关卡选择（不影响灵宠池，无惩罚）

### 关卡结算

| 评价 | 条件 | 额外奖励 |
|------|------|----------|
| S | ≤5 回合通关且满血 | 碎片 ×2 + 高级宠掉率翻倍 |
| A | ≤8 回合通关 | 碎片 ×1.5 |
| B | 通关 | 基础碎片 |

### 新增模块

| 新增文件 | 职责 |
|----------|------|
| `js/data/stages.js` | 关卡配置数据 |
| `js/engine/stageManager.js` | 关卡生命周期（选关→编队→战斗→结算） |
| `js/views/stageSelectView.js` | 关卡选择 UI |
| `js/views/stageTeamView.js` | 编队 UI |
| `js/views/stageResultView.js` | 关卡结算 UI |

### 对现有代码的改动

| 改动 | 涉及文件 | 说明 |
|------|----------|------|
| 战斗引擎适配 | `js/engine/battle.js` | 新增 `g.battleMode` 字段（`'roguelike'` \| `'stage'`），结算逻辑按模式分支 |
| 标题页入口 | `js/views/screens.js` | 灵宠池 ≥ 5 只时显示"固定关卡"按钮 |
| 场景注册 | `js/main.js` | 新增 stageSelect/stageTeam 场景的 render/touch 分发 |
| 存储扩展 | `js/data/storage.js` | 新增 `stageClearRecord`（首通记录）字段 |

## Phase 4：高级灵宠设计（建议 2 周）

### 设计概要

- 高级灵宠与普通灵宠共用基础框架（属性/攻击/技能/星级）
- 区别：满星 ★4、数值更高、★4 解锁终极形态
- 首批设计 15~20 只高级灵宠（每属性 3~4 只）

### 数值档位

| 星级 | 普通灵宠攻击倍率 | 高级灵宠攻击倍率 |
|------|-----------------|-----------------|
| ★1 | ×1.0 | ×1.0 |
| ★2 | ×1.3 | ×1.3 |
| ★3 | ×1.7 | ×1.7 |
| ★4 | - | ×2.2 |

### ★4 终极形态

终极形态在 ★3 技能 A 强化的基础上，额外获得以下之一：

| 终极类型 | 说明 |
|----------|------|
| 双技能 | 解锁独立的技能 B（独立 CD） |
| 终极被动 | 永久被动效果（如"每回合自动回血 3%"） |
| 属性觉醒 | 大幅强化属性加成（如"金属性伤害 +30%"） |

### 代码改动

| 改动 | 涉及文件 |
|------|----------|
| 宠物星级扩展 | `js/data/pets.js`（`getPetStarAtk` 增加 ★4 倍率；新增 `skill4` 字段） |
| 技能引擎适配 | `js/engine/skills.js`（`triggerPetSkill` 支持双技能 / 被动检测） |
| 战斗引擎适配 | `js/engine/battle.js`（被动效果在回合开始时自动触发） |
| 高级灵宠数据 | `js/data/pets.js` 或新增 `js/data/advancedPets.js` |
| 灵宠池 UI 适配 | `js/views/petPoolView.js`（展示 ★4、终极形态标记） |

## Phase 5：多波次战斗 + 捕捉系统（建议 3~4 周）

> 此阶段在固定关卡成熟后推进。

### 多波次战斗

- 关卡配置 `enemies` 改为 `waves` 数组
- 每波击败后短暂过场（展示"第 X 波"），棋盘保留、buff 保留
- 最终波击败后进入结算

```javascript
waves: [
  { enemies: [{ ... }], isBoss: false },
  { enemies: [{ ... }], isBoss: false },
  { enemies: [{ ... }], isBoss: true },
]
```

### 捕捉系统

| 设计点 | 方案 |
|--------|------|
| 可捕捉标记 | 关卡配置中标记 `capturable: true` 的敌人 |
| 捕捉条件 | 敌人血量 ≤ 20% 时出现"捕捉"按钮 |
| 捕捉成功率 | `基础概率(30%) + 残血加成(血量越低概率越高) + 道具加成` |
| 捕捉失败 | 不消耗回合，但按钮进入冷却（2 回合后可再尝试） |
| 捕捉成功 | 敌人消失，战斗继续（多波次中该敌人视为击败）；★1 入灵宠池 |
| 捕捉道具 | 新增消耗品"灵缚符"，使用后本次捕捉概率 +30% |

### 代码改动

| 改动 | 涉及文件 |
|------|----------|
| 多波次流程 | `js/engine/stageManager.js`（波次切换逻辑） |
| 战斗状态扩展 | `js/engine/battle.js`（`bState = 'waveTransition'`；victory 时检查是否还有下一波） |
| 捕捉逻辑 | `js/engine/battle.js`（新增捕捉判定函数） |
| 捕捉 UI | `js/views/battleView.js`（敌人残血时渲染捕捉按钮） |
| 捕捉触摸 | `js/input/touchHandlers.js`（战斗中捕捉按钮点击处理） |

## Phase 6：灵宠合成（建议 2 周）

### 设计概要

- 两只灵宠合成一只新灵宠
- 合成有一定规则可循（非完全随机）：
  - 同属性合成：大概率出同属性更高档位灵宠
  - 跨属性合成：有概率出特殊灵宠
- 保底：合成结果品质 ≥ 两只原料中较低的
- 合成前显示"可能获得"范围（如"??? 金属性灵宠 T2~T3"）

### 合成规则（初版）

| 原料组合 | 结果范围 |
|----------|----------|
| 普通 + 普通（同属性） | 同属性普通灵宠（档位 +1 概率 60%） |
| 普通 + 普通（跨属性） | 任一原料属性灵宠 80%，特殊灵宠 20% |
| 普通 + 高级 | 高级灵宠（同属性概率高） |
| 高级 + 高级 | 高级灵宠（档位 +1 概率 40%，特殊灵宠概率 30%） |

### 代码改动

| 改动 | 涉及文件 |
|------|----------|
| 合成逻辑 | 新增 `js/data/fusion.js`（合成配方、概率计算） |
| 合成 UI | 新增 `js/views/fusionView.js`（选择原料、预览、确认） |
| 灵宠池联动 | `js/data/storage.js`（合成后移除原料、添加结果） |

## Phase 7+：远期规划

### 特殊棋盘模式

- **不改动五行基础系统**
- 在固定关卡中引入特殊规则变种：
  - **双属性关卡**：棋盘只有 2~3 种属性珠（数量更多，更易触发高 Combo）
  - **小棋盘速战**：4×5 棋盘，限时 5 秒，考验瞬间判断
  - **大棋盘混战**：6×7 棋盘，多目标同时在场
  - **特殊珠子**：毒珠（消除后对敌施加 DOT）、虹珠（可匹配任意属性）

### 社交系统

- 好友灵宠池互相查看
- 好友灵宠借用（固定关卡中借用好友一只灵宠替代空位）
- 灵宠交换

### 赛季系统

- 每赛季（月度/季度）重置排行榜
- 赛季专属关卡和限定灵宠
- 赛季结算奖励
